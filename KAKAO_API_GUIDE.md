## ✅ 카카오 API 키 종류

카카오 개발자 센터에서는 4가지 API 키를 제공합니다:

| 키 이름 | 용도 | 우리 프로젝트에서 사용? |
|--------|------|----------------------|
| **네이티브 앱 키** | Android/iOS 앱 | ❌ 사용 안 함 |
| **REST API 키** | 서버 측 API 호출 | ✅ **이것 사용!** |
| **JavaScript 키** | 웹 브라우저에서 호출 | ❌ 사용 안 함 |
| **Admin 키** | 관리자 기능 | ❌ 사용 안 함 |

**중요:** 우리는 **REST API 키**만 사용합니다!

## 🔑 API 키 설정 방법

### Step 1: `.env.local` 파일 생성

프로젝트 루트(`E:/lunchmenu`)에 `.env.local` 파일을 만들고:

```env
# 기상청 API (필수)
NEXT_PUBLIC_KMA_API_KEY=여기에_기상청_서비스키_입력

# 카카오 REST API 키 (필수)
NEXT_PUBLIC_KAKAO_MAP_API_KEY=f53f2bae4d1a02bd9c49a99591d3034b
```

### Step 2: 서버 재시작

```bash
# Ctrl+C로 서버 중지
npm run dev
```

## 🔧 API 호출 예시

우리 프로젝트에서는 이미 구현되어 있습니다!

**파일 위치:** `/api/restaurants/route.js`

**호출 방식:**
```javascript
const response = await fetch(
  `/api/restaurants?query=짜장면&lat=37.5665&lon=126.9780&radius=1000&sort=distance`
)
```

**파라미터:**
- `query`: 검색할 메뉴명 (예: "짜장면", "치킨")
- `lat`: 위도
- `lon`: 경도
- `radius`: 검색 반경 (미터, 기본값: 1000m)
- `sort`: 정렬 방식 (`distance`: 거리순, `accuracy`: 정확도순)

## 📱 작동 확인

### 1. 서버 시작
```bash
npm run dev
```

### 2. 브라우저에서 테스트
1. 메뉴 추천받기
2. "근처 가게 찾기" 버튼 클릭
3. 개발자 도구(F12) → Console 탭 확인

### 3. 정상 작동 시
- 실제 가게 목록이 콘솔에 출력됨
- 추후 UI로 표시 예정

### 4. API 키 없을 때
- "가게 검색 기능은 준비 중입니다!" 알림 표시
- 모의 데이터 5개 반환
